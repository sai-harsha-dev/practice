THEORY
-------
- Kubectl is a command line interface which interacts with the API servers REST API endpoint in the mater nodes.
- The cluster configuration are maintained in the configuration file present in users/.kube/confg.

COMMANDS
---------
1) kubectl version --short  ===> shows the version client and server (must vary only by +-1 not more than that)

(* rememeber for the cluster to show up on docker desktop update configuration in .kube\confg file under user pofile)


2) kubectl get nodes ===> show the details of the nodes in the cluster
